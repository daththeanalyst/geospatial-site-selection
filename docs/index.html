Project Overview

This project identifies optimal locations for new retail businesses across all 33 boroughs of Greater London using multi-modal geospatial data and supervised machine learning. The model supports 6 business types (cafe, restaurant, pub, fast food, gym, bakery). The core insight: a binary classifier's False Positives — locations the model predicts should have a given business but currently don't — represent untapped market opportunities.

In the language of network theory, these are Burt's Structural Holes (1992): positions in a spatial network with high demand signals but no existing supply, now identified by data-driven learning rather than heuristic scoring.

Step 01: Data Pipeline (ETL)

The first notebook acquires source datasets from three geospatial modalities and harmonises them into a common coordinate reference system (EPSG:27700, British National Grid).

Three Data Modalities

ModalityRepresentationExamplePrecision
Vector Discrete geometric primitives with exact coordinate pairs OSM cafe locations (Points), building footprints (Polygons) Sub-metre
Raster Regular grid of cells, each storing a numeric value LandScan population grid — each pixel holds a count ~1 km per pixel

POI Categorisation

Each Point of Interest is classified into a business role relative to the target business type:

RoleOSM Tags (examples)Effect on Score
CompetitorSame type as target (e.g. cafes for cafe model)Excluded from features (encodes the target)
Synergygym, university, office, library, co-workingIncreases score (complementary foot traffic)

CRS Fix Applied: The original code computed centroids in EPSG:4326 (degrees). At 51°N, 1° longitude spans ~69 km while 1° latitude spans ~111 km — a 60% distortion. All geometric operations now use EPSG:27700 (metres) to eliminate this anisotropic error.

Competition Density (k=2 Ring)

For each business type, a competition density feature counts same-type businesses in a k=2 H3 ring (~18 neighboring hexagons, ~350m radius). This wider radius ensures hexagons over open spaces (parks, railway corridors) can still "see" nearby businesses on surrounding streets, preventing artificial dead zones in the feature matrix.

Census Merge

Three ONS Census 2021 CSVs (Economic Activity, Age Structure, Qualifications) are merged on geog_code (Output Area identifier), producing 846 OAs with 7 demographic columns. Missing values are imputed with the column median (not zero, which would falsely imply "0% employed").

Step 02: H3 Hexagonal Grid & Enrichment

Uber's H3 library partitions Greater London into hierarchical hexagonal cells. Unlike square grids (which have ambiguous adjacency at corners), hexagons have 6 equidistant neighbours — ideal for walking-distance analysis.

Why Resolution 9?

ResolutionEdge LengthHex AreaUse Case
7~1.2 km~5.16 km²District-level
8~460 m~0.74 km²Neighbourhood
9~174 m~0.105 km²Walking-scale (our choice)
10~66 m~0.015 km²Street-level

At Resolution 9, each hexagon covers roughly the area a person walks in 2 minutes — the micro-unit of the 15-minute city (Moreno et al., 2021).

Census demographics are joined via sjoin_nearest with population-weighted mean aggregation: larger Output Areas contribute proportionally more to each hex's demographic profile. Hexes with no OA match receive the borough median (conservative imputation).

Step 03: Graph Analytics & Heuristic Scoring

The enriched H3 grid becomes a spatial graph where hexagons are nodes and edges connect adjacent hexes, weighted by inverse distance in metres. From this graph we extract three layers of structural information: six centrality metrics (individual node importance), Louvain community detection (meso-scale neighbourhood clusters), and Node2Vec embeddings (latent structural roles learned via random walks).

Six Centrality Metrics

MetricWhat It MeasuresBusiness Meaning
DegreeNumber of neighboursInterior (6) vs boundary (<6) connectivity
BetweennessFrequency on shortest pathsTransit corridor importance
ClosenessAverage distance to all nodesCentral vs peripheral location
ClusteringNeighbour interconnectionNeighbourhood cohesion
EigenvectorInfluence via well-connected neighboursHub proximity (near other hubs)
PageRankDamped random walk steady stateFoot-traffic accumulation potential
Weight Semantics: NetworkX treats the weight parameter differently across algorithms. Path-based metrics (betweenness, closeness) interpret weight as cost to minimise, so we use actual Euclidean distance. Clustering interprets weight as connection strength, so we use the IDW value (1/(d+1)).

Louvain Community Detection

Beyond individual centrality scores, the spatial graph encodes meso-scale structure — clusters of hexagons that are more densely connected to each other than to the rest of the network. We apply the Louvain algorithm (Blondel et al., 2008) to partition the H3 adjacency graph into communities, maximising modularity (the fraction of edges falling within groups minus the expected fraction if edges were placed at random).

Each hexagon receives a community_id (integer label) that captures which spatial neighbourhood cluster it belongs to. This allows the ML model to learn that certain community clusters share latent characteristics — for example, a community covering a town centre with mixed retail may behave differently from one covering a residential suburb, even if their individual centrality scores are similar.

Node2Vec Graph Embeddings

Centrality metrics and community labels capture explicit structural properties, but the graph also contains implicit patterns that are difficult to express as hand-crafted features. Node2Vec (Grover & Leskovec, 2016) learns a low-dimensional vector representation for each hexagon by simulating biased random walks on the graph, then training a Word2Vec skip-gram model on the resulting "sentences" of node visits.

We generate 3 embedding dimensions (node2vec_dim0, dim1, dim2) per hexagon. These capture latent structural roles — hexagons with similar graph neighbourhoods (e.g., inner-ring transit corridors, suburban cul-de-sacs) receive similar embedding vectors, even if they are geographically distant. The walk parameters (p=1, q=1) balance between local neighbourhood exploration and broader structural equivalence.

Why 3 dimensions? Adding more dimensions risks overfitting on a graph with ~16k nodes and relatively uniform H3 adjacency. Three dimensions capture the dominant structural variation (urban core vs. suburb vs. boundary) while keeping the feature matrix compact.

Heuristic Site Score

ScoreH = PopH × DH + 5 · SH + 3 · AH − 15 · CH

Where DH = Level 4 qualification % / 100 (demand proxy), S = synergy count, A = anchor count, C = competitor count. This is an interpretable heuristic — the ML model learns optimal weights from data.

3D Visualisation

Scores are normalised to [0, 1] using min-max scaling, then mapped to a green (high) → red (low) colour gradient. Extrusion height is proportional to the normalised score. The final 3D map is generated automatically by the notebook and embedded in the Results tab.

Machine Learning Model

Problem Formulation

Binary classification: for each of 6 business types, predict whether a hexagon should contain that business based on its geospatial features. The commercially valuable output is the False Positives — locations predicted as suitable that have no current supply.

Leakage Guard: For each business type model, n_{type} (the count of that type) is excluded from the feature matrix because it directly encodes the target variable. All other POI type counts are retained as co-occurrence features, enabling the model to learn which business types tend to cluster together.

Spatial Cross-Validation

Standard k-fold CV violates spatial independence (Tobler's First Law). Adjacent hexes leak similar features into both train and test sets, inflating AUC by 5–15%.

Solution: Group Resolution-9 hexes by their Resolution-7 parent cell (~0.74 km² blocks). All hexes sharing a parent are assigned to the same fold via greedy class-balanced assignment, preserving spatial integrity and guaranteeing both classes in every fold across 5 folds.

Threshold Tuning (F1)

The default classification threshold of 0.5 is rarely optimal for imbalanced datasets. We use Precision–Recall curves on honest out-of-fold predictions from spatial CV to find the threshold that maximises F1 per business type.

F1 balances precision and recall equally, ensuring the model captures most existing businesses while still maintaining high-confidence recommendations — each green hexagon on the map represents a genuine opportunity, and the map avoids excessive blind spots.

Why not 0.5? At the default threshold, the model over-predicts — producing hundreds of False Positives including many borderline cases. The F1-optimal threshold balances precision and recall for credible map coverage with fewer blind spots.

All gaps fall below the 0.10 concern threshold. Gym shows the largest gap (0.076) due to having the fewest positive examples (~3.2% positive rate), which gives the model less signal to learn from and more room for memorisation. This is expected behaviour for rare-class problems, not a defect.

2. Feature Ablation (Competition Density)

The nearby_{type} feature counts same-type businesses in a 2-ring neighbourhood (~350m). This could theoretically leak target information. To test, we re-run each model with this feature dropped and measure the AUC change:

All deltas were < 0.002 AUC for 5 of 6 types, confirming competition density is not driving the model’s predictions. Gym showed a +0.015 improvement when dropped, suggesting this feature adds slight noise for the rarest class. Verdict: no soft leakage detected.

3. Learning Curve

We train on increasing fractions of the data (20%, 40%, 60%, 80%, 100%) and track both train and test AUC. A converged learning curve (flat at full data) means the model has extracted all available signal and more data would yield diminishing returns.

The last increment was +0.0001 AUC — the model has fully converged. Train and test curves approach each other as data increases, the classic signature of a well-regularised model.

Results & Recommendations

The False Positive Thesis

Key Insight: False Positive hexagons possess all the learned features of successful business locations — high footfall, educated demographics, strong transit connectivity, co-located complementary businesses — but no one has opened that business there yet. These are Burt's Structural Holes, validated by supervised learning. The F1-optimal threshold balances precision and recall for credible map coverage.
